📌 Classification Methods
Scenario triggerMethod
Same number of features per classQuantile
Visually balanced map / show relative rankQuantile
Skewed data / outliers / natural clustersNatural Breaks
Show true magnitude differencesNatural Breaks
Above/below average / normally distributedStd Deviation
Uniform / evenly spread distributionEqual Interval
Map series / cross-time comparisonEqual Interval (fixed breaks)
Know each weakness
Quantile → groups unlike values together  |  Equal Interval → empty classes with skewed data  |  Natural Breaks → NOT comparable across maps  |  Std Deviation → misleading with non-normal data
Trap to watch
"Normally distributed data — put equal features per class" → the normal distribution is a red herring. Equal features per class = Quantile regardless of distribution shape.
🗃️ Raster Operations — Local / Focal / Zonal / Global
Local

Cell-by-cell math across rasters. No neighbors. No zones. Matching positions only.

Add/subtract/multiply rasters · reclassify values · change detection (raster A − raster B)

Focal

Each cell calculated from its surrounding neighborhood / moving window.

Slope · aspect · smoothing filter · majority filter · any "neighboring cells" scenario

Zonal

Statistics calculated per zone / polygon group.

Avg elevation per watershed · max temp per climate zone · total precip per county

Global

Uses all cells in the entire raster for each output cell.

Euclidean distance · cost distance · viewshed analysis

Quick test
Neighbors involved? → Focal  |  Per zone/polygon? → Zonal  |  Whole raster? → Global  |  Cell-to-cell math? → Local
🔀 Overlay Operations — Union / Intersect / Clip / Erase
OperationOutputTrigger words
UnionEverything from A + B, full extent, all attributes"all features from both," "combine," "merge"
IntersectOnly where A and B overlap, both attributes"within," "inside," "overlap," AND condition
ClipA cut to B's shape, only A's attributes"cut to boundary," "extract within study area"
EraseA minus B's area"outside," "beyond," "more than X away," "exclude"
Direction rule — most important
Want what's INSIDE a zone → Buffer → Intersect
Want what's OUTSIDE a zone → Buffer → Erase
"Forest more than 2km from roads" → Buffer roads → Erase buffer from forest

Attribute accuracyFeature location is correct but attribute value is wrongHighway coded as residential street

Two-question test
Wrong place? → Positional  |  Features contradict? → Logical  |  Wrong value? → Attribute  |  Out of date? → Temporal
Intentional vs unintentional error
Intentional: generalization · map projection distortion (cartographer chooses to accept these)
Unintentional: GPS error · digitizing mistakes · scanner distortion

🗺️ Map Types
Map typeData requirementTrigger
ChoroplethNormalized only rates, %, density — NEVER raw counts"per capita," "per sq mile," "rate," "percentage"
Graduated symbolRaw counts — single scaled symbol per location"symbol size proportional to value"

Most-tested rule
Choropleth must NEVER use raw counts. Larger areas always have more of anything — normalize first (divide by population, area, etc.)
🔗 Topology
Topology enables — memorize all 3
1. Selecting adjacent features    2. Enforcing planar data integrity    3. Automating data clean-up
RuleApplies toFlags
Must not overlapPolygonsTwo polygons sharing interior space
Must not have gapsPolygonsUncovered sliver between polygons
Must not have danglesLinesUnconnected line endpoint
Must not have pseudo nodesLinesOnly two lines meeting end-to-end (no true intersection)
Planar enforcementPolygonsNo gaps + no overlaps = every location in exactly one polygon
Topology does NOT
Increase positional accuracy — topology works with coordinates you already have. It enforces relationships, not precision.

Horizontal datums

WGS 84 (GPS default · global)
NAD 83 (US standard · GRS 80)
NAD 27 (legacy · Clarke 1866)
NAD 27→83 shift = up to 100m — EXPECTED

Vertical datums

NAVD 88 = current US standard elevation
NGVD 29 = legacy (convert to NAVD 88)
GPS → ellipsoidal height
GEOID18 → converts to NAVD 88
NADCON = horizontal transform only

State Plane vs UTM
State Plane: ~3° wide zones · Lambert (wide states) or Transverse Mercator (tall states) · WA = Lambert
UTM: 6° wide · 60 zones · WA = zones 10 & 11 · False easting 500,000m · Scale factor 0.9996 (secant)
Easting math: Easting − 500,000 = distance from central meridian (+ = east, − = west)

Agile: iterative cycles · continuous feedback · frequent delivery · requirements evolve

📐 Analytical Methods
MethodUse case
BufferZone of specified distance around features. Output = always a vector polygon.
DissolveMerge features sharing the same attribute value. Removes internal boundaries.
Spatial joinTransfer attributes based on geographic location/relationship
Point-in-polygonDetermine which polygon each point falls within — transfers polygon attributes to point
KDESmooth continuous density surface from points. Bandwidth controls smoothness.
KrigingGeostatistical interpolation using spatial autocorrelation. Gold standard for unknown values.
IDWInverse Distance Weighting — closer points weighted more. Simpler than Kriging.
Thiessen / VoronoiEvery location assigned to nearest point feature. Proximity-based, not statistical.
Cost pathLeast-cost route across a cost surface raster (pipeline routing, corridor siting)
Watershed delineationStarts with DEM → fill sinks → flow direction → flow accumulation → pour point → boundary
Map algebra operations
Local: cell × cell (multiply slope × erodibility)  |  Logical: cells where value > 50 → returns 1/0 binary raster  |  Mathematical: square root, log of cell values

ConceptDefinition

Active sensorEmits own energy, measures return · works day/night/cloud cover · LiDAR (laser) · SAR/RADAR (microwave)
Spatial resolutionPixel size on the ground (1m vs 30m)
Spectral resolutionNumber and width of wavelength bands captured
Temporal resolutionHow frequently sensor revisits same area
Radiometric resolutionSensitivity to signal differences — expressed in bits (8-bit = 256 values)
NDVI(NIR − Red) / (NIR + Red) · healthy vegetation = high NIR, low Red = high NDVI
LiDARTime-of-flight laser · multiple returns (canopy, branches, ground) · stored as LAS/LAZ
Vertical imageryNadir (straight down) · accurate measurements · photogrammetric mapping · consistent scale
Oblique imageryAngled view · shows building facades · better urban context visualization
🧭 Coordinate Systems Quick Reference
SystemUnitsUse
GCS (Geographic)Degrees (angular)Storage format · global data · cannot measure distance directly
PCS (Projected)Meters or feet (linear)Analysis · distance/area measurement · flat surface
State PlaneFeet (US)County/state-level precision work · surveying · cadastral
UTMMetersRegional mapping · 6° wide zones · WA = 10 & 11
Web MercatorMetersWeb basemaps only · distorts area · NOT for analysis
North America coordinate check
Longitude must be NEGATIVE (−60° to −170°) = west of prime meridian
Latitude must be POSITIVE (+15° to +85°) = north of equator
Positive longitude = eastern hemisphere = NOT North America
.prj file WKT check
GEOGCS = Geographic Coordinate System (degrees)  |  PROJCS = Projected Coordinate System (linear units)
No .prj file = unknown coordinate system — do not assume WGS 84
✅ Select All That Apply — Test Strategy
The habit to build
Treat every option as an independent true/false question. Don't look for THE answer — evaluate each option on its own. Every option that is independently true must be selected. Never stop early because you think you have enough.
Red flag words — almost always WRONG
"always" · "never" · "only" · "completely replaced" · "eliminates the need for" · "cannot be used with"
Absolute statements are wrong ~80% of the time on this exam.
Common traps
• Topology increases positional accuracy ❌ — it enforces relationships, not precision
• GeoJSON supports multiple coordinate systems ❌ — WGS 84 only
• VGI is always less accurate than professional data ❌ — "always" = red flag
• Waterfall = iterative cycles ❌ — that's Agile
• Natural Breaks is comparable across maps ❌ — recalculates each time
